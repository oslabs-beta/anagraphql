Why AnagraphQL?

A majority of API providers aren't using GraphQL. Why? With RESTful APIs, the amount of data returned in a request is fixed per endpoint and easy to price. On the other hand, GraphQL queries return the exact data that clients are seeking and queries may be different each time. This can cause headaches when it comes to pricing structures. AnagraphQL solves this by enforcing policies on GraphQL queries, allowing the API provider to control data flow.
